## 1. Identidade do Produto

**O que é:** App mobile que mostra, em 10 segundos, onde o dinheiro do usuário rende mais entre bancos e corretoras brasileiras (XP, BTG, Rico, Inter, C6).

**Público-alvo:** Classe média brasileira, 25-45 anos, smartphone Android ou iOS, alguma renda disponível para investir, mas sem familiaridade com jargão financeiro. Muitos têm analfabetismo funcional parcial — leem, mas têm dificuldade com textos densos ou abstratos.

**Promessa central:** Clareza brutal. O usuário nunca deve se perguntar "o que isso significa?".

## 3. Design System

> Detalhes completos em `references/design-system.md`

### 3.1 Filosofia

- **Referência:** Nubank (clareza, confiança, hierarquia tipográfica forte) + Robinhood (foco no número principal)
- **Diferencial:** Verde no lugar do roxo. Light mode exclusivo.
- **Princípio #1:** O número mais importante deve ser o elemento mais visualmente dominante da tela.
- **Princípio #2:** Se o usuário precisar ler mais de 2 linhas para entender uma ação, o design falhou.
- **Princípio #3:** Nunca mostre jargão sem tradução imediata. "CDI" sempre acompanha "rendimento ao ano" ou equivalente plain language.

## 4. Tom de Voz e Copywriting

> Detalhes e exemplos completos em `references/copywriting.md`

### 4.1 Princípios

1. **Fale como gente.** "Você pode sacar quando quiser" em vez de "Liquidez D+0".
2. **Mostre o número antes da explicação.** "Rende 13,2% ao ano — mais que a poupança."
3. **Nunca culpe o usuário.** Erros de formulário: "Hmm, esse CPF não tá batendo. Confere?" não "CPF inválido".
4. **Seja breve.** Se um botão pode ter 2 palavras, não use 4.
5. **Jargão permitido com tradução.** "CDI (rentabilidade de referência do mercado)" na primeira aparição de cada sessão.

### 4.2 Glossário de Traduções Obrigatórias

| Jargão técnico | Plain language (usar no UI) |
|---|---|
| CDI | quanto rende por ano |
| D+0 / Liquidez imediata | saque quando quiser |
| D+1 | disponível amanhã |
| D+30 | disponível em 30 dias |
| No vencimento | só no final do prazo |
| FGC coberto | dinheiro protegido pelo governo até R$250 mil |
| CDB | tipo de investimento em banco |
| LCI / LCA | investimento isento de imposto |
| % do CDI | quanto rende comparado ao padrão do mercado |
| Score de recomendação | pontuação de indicação |

### 4.3 Mensagens de Estado

```
Loading:        "Buscando as melhores taxas..."
Empty state:    "Não encontramos opções para o seu perfil agora. Tente mudar os filtros."
Error network:  "Sem conexão. Mostrando os dados mais recentes."
Error server:   "Algo deu errado do nosso lado. Já estamos vendo isso."
Success click:  "Você vai ser redirecionado para abrir sua conta."
Outdated data:  "Taxas de [hora]. Puxe para atualizar."
```

## 9. Animações e Micro-interações (Moderado)

```
USE Reanimated 3 APENAS para:
  1. Entrada de cards (FadeInDown escalonado)
  2. Número do score animado (contagem de 0 até valor real, 600ms)
  3. Skeleton pulse (opacity loop)
  4. Transição de slides do onboarding (slide horizontal)
  5. Botão pressed state (scale 0.98)

NÃO USE animação em:
  - Transições de tab (padrão do Expo Router)
  - Toasts / alerts (padrão do sistema)
  - Ícones (estáticos)

Haptics (expo-haptics):
  - impactAsync('light'): tap em pills/filtros
  - impactAsync('medium'): tap em botão primário "Investir"
  - notificationAsync('success'): após redirect para afiliado
  - notificationAsync('error'): erro de rede

Skeleton Loading:
  - Mostrar SEMPRE antes dos dados (nunca tela em branco)
  - TTL de exibição mínimo: 300ms (evita flash se dados vêm rápido)
  - Fade-out suave quando dados chegam: opacity de 1 → 0 em 200ms
```

## 12. Acessibilidade (obrigatório)

```
- accessibilityLabel em TODOS os Pressable, TouchableOpacity e botões
- accessibilityHint quando a ação não é óbvia pelo label
- accessibilityRole: 'button' em elementos clicáveis que não são <Button>
- Contraste mínimo: 4.5:1 para texto normal, 3:1 para texto grande
  → brand-500 (#16A34A) sobre white (#FFF): ratio 4.54:1 ✓
  → neutral-400 (#A3A3A3) sobre white: ratio 2.32:1 ✗ — usar apenas para placeholder
- Tamanho mínimo de toque: 44x44px (mesmo que visualmente menor)
- Não dependa SOMENTE de cor para transmitir informação (use ícone + cor)
- Suporte a Dynamic Type (iOS): use unidades relativas, nunca fontSize fixo sem escala
```

## 13. Performance

```
- FlatList com getItemLayout definido (evita jumping no scroll)
- keyExtractor: sempre usar ID único do banco, nunca index
- Imagens de logo: cache via expo-image (não Image do RN)
- TanStack Query:
    staleTime: 15 * 60 * 1000  (15 minutos)
    cacheTime: 30 * 60 * 1000  (30 minutos)
    retry: 2
- Supabase Realtime: apenas na tela Home, desconectar ao sair (useEffect cleanup)
- Bundle size: não importar library inteira. Ex: import { X } from 'phosphor-react-native', não import *
```

## 14. Regras de Ouro (não negociável)

1. **Lógica financeira fora do componente.** Toda função de cálculo vai em `/lib/`. Componente só renderiza.
2. **TypeScript strict. Nunca `any`.** Se não sabe o tipo, modelar corretamente.
3. **Toda função financeira tem teste Jest.** `calculateProjectedReturn`, `scoreOffer`, `formatBRL`, etc.
4. **Sem credencial no código.** Supabase URL e anon key em `.env` via `expo-constants`.
5. **Plain language primeiro.** Qualquer texto novo: revisar pelo glossário (seção 4.2). Jargão só com tradução.
6. **Componente > 200 linhas → quebrar.** Sem exceção.
7. **MVP primeiro.** Não implementar feature P2 enquanto P0 não estiver em produção.
8. **Botão de afiliado sempre verde, sempre visível no card.** É a principal fonte de receita.
